Reset `event-context.md` to blank template, then run session-init to configure a new event.

## Steps

1. Read `event-context.md`. If it is already populated (no `[TBD]` markers), confirm with the user: "Current event '<name>' will be discarded — continue?" Abort on anything but explicit yes. If already blank, skip confirmation.
2. Overwrite `event-context.md` with the blank template below.
3. Delegate immediately to the `session-init` agent.

Blank template to write:

```
# Event Context — AGEVP

**Name:** [TBD]
**Date:** [TBD]
**Type:** [festival / sports / cultural / workshop / gala / other]
**Expected attendance:** [TBD]
**Fixed budget:** €[TBD]
**Event lead:** [TBD]
**Key contacts:**
- [Name] — [Role]

**Preferred area:** [Paris — arrondissement or neighbourhood]
**Constraints / notes:** [TBD]

---
*Generated by session-init — not versioned*
```

## Rules

- Never discard a populated event context without explicit user confirmation.
- After writing the template, launch session-init without further prompting.
